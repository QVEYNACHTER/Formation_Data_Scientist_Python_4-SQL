Pour ce projet, on m'a chargé de trois missions : Implémenter des requêtes SQL, aboutir à une segmentation des clients que les équipes d'e-commerce pourront utiliser au quotidien pour leurs campagnes de communication, et simuler un contrat de maintenance. Les requêtes SQL sont dans le premier livrable, les trois notebooks concernent les deux autres missions. Dans le premier, j'utilise la librairie sqlite3 pour préparer les données via des requêtes SQL directement dans le notebook, procède à un rapide nettoyage des données et une courte analyse exploratoire. Dans le second, je teste trois modèles de clustering différents : k-means, CAH, DBSCAN pour les comparer entre eux et retenir celui qui fait le plus de sens. Enfin dans le dernier notebook, je simule l'évolution de la stabilité du clustering dans l'optique de faire une recommandation de fréquence à laquelle la segmentation doit être mise à jour pour rester pertinente. Pour ce faire : Le jeu de données allant de 01-2017 à 08-2018, je décide de partir d'une période de base qui contient les dernières commandes passées la première année. Ensuite, pour mesurer la stabilité dans le temps, j'injecte progressivement les dernières commandes passées entre 01-2018 et 08-2018 en recalculant à chaque fois l'indice de Rand (ARI) ajusté par rapport aux données de la période de base. Je procède à un test par mois, puis par semaine et conclue par une analyse de l'évolution de la distribution des features (lorsque c'est pertinent).
